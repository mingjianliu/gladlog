## 2. Layer B 为什么还没跑

`calibrate-judge.md` 规定 **5/7 维通过才能判分**。n=10 套件(80 件,seed 42)结果:

| ------------------------- | --------------- |

| outcomeAlignment 80%      | labelBias 70%   |
| focusCalibration 80%      |                 |

**4/7。** 但三个 FAIL 性质不同,这才是要点:

### 阻塞 A —— accuracy 判官间方差 ±2(BACKLOG 14.5)

`noise` 与 `labelBias` 的失败**全部是特异性,不是敏感性**:判官对目标缺陷压得又准又狠
(5→3、5→1),但 accuracy 同时漂移 2 分,超过 ±1 容差被判失败。

方差是实测的,不是推论。做法:`none` / `severity-labels` / `duplicated-noise` 三类扰动
**都不删内容、不改 response**,所以同一源的这三件 = **同一份材料被三个独立判官读三遍**。
测得 accuracy 极差均值 **1.00**、10 源里 4 个 ≥2。

**所以 noise/labelBias 的 FAIL 是 accuracy 方差的投影,不是它们自己的缺陷。**

### 阻塞 B —— sufficiency 真盲区(BACKLOG 14.2)

跟方差无关:10 例里 **6 例 `5→5`** —— prompt 里死亡行全部删光,判官一分不扣。纯敏感性
失败,n=5 两轮 + n=10 一轮**三次独立测量都复现**。BACKLOG 里两个方向都没做。

### 怎么测(判据已预先登记,别改)

1. 重评 **30 件**:`judge-calibration/cases/` 里 10 个源 × `{none, severity-labels, duplicated-noise}`,
   输出到 `scores-det2/`(**已有 5 件**:case-01/06/08/13/14)。
2. 判据:**每个源三件的 accuracy 极差**,取均值 / 最大 / ≥2 的源数。
3. 基线:v1 自选 3 条 = **1.00**,v2 规则集 = **0.80**。

### ⚠ 读数时的陷阱(已有 5 件的数据暴露出来的)

查表锚点把「恰 1 处小错」从 3 改成 4。已回来的 5 件里**四件从 3 变 4**,纯粹是映射上移,
判官的实质分歧一点没变。源 8 的 none/dup 配对极差从 2 变 1,也是这么来的 ——
一个判官找到 0 个错、另一个找到 1 个,**分歧照旧**,只是刻度变细把差距压成了 1 格。

**所以极差下降不等于方差下降。** 必须同时比对**判官找到的错误集合是否一致**
(`factAudit` 里 verdict ∈ refuted/unsupported 的条目),那才是实质指标。建议把这个
改成主判据、极差降为辅助判据。

### 机制拆分(已做完,直接用)

对 `scores-det` 逐案分析,修后仍有分歧的 5 个源分成两类:

| 机制             | 源      | 极差  | 表现                                         |
| ---------------- | ------- | ----- | -------------------------------------------- |
| **(a) 查证漏检** | 1, 4, 8 | **2** | 一个判官发现错,另两个判 verified             |
| **(b) 锚点映射** | 2, 6    | 1     | 三个判官找到**同一个错**,给出 3/3/4 与 3/4/4 |

**主因是 (a),不是 (b)。** 关键发现:漏检的**不是没抽到那条主张**。源 4 的
「41%(0:31)→21%(0:33)」带时间戳、必然在规则集内,三个判官都审计了它,两个查出来
一个判 verified。漏掉的错几乎全是数字对不上:`41%` vs `42%`、`0:31` vs `0:32`、
`19% 最低` vs 实际 `16%`、「只有三次 Drink」实际还有 trinket 与 Angelic Feather。

`3d92ba3` 的两条正是分别针对 (a) 和 (b) 的。
